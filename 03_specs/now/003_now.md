# 003 - Refactorizacion email-first para Knowledge Transfer / Interreg

> Fecha: 2026-06-07
> Status: done
> Input aprobado: refactorizacion global del enfoque de scraping para que el repo opere como motor de prospeccion B2B email-first

## Outcome

Corregir el contrato estrategico y documental del repositorio para que:

- el scraping se mida por su capacidad de producir leads accionables para outreach;
- ninguna organizacion se considere completa sin `email` o `email_status`;
- `skilland-knowledge-transfer-engine`, y en especial `interreg_mac`, deje de presentarse como output final scored y pase a presentarse como capa intermedia antes del enriquecimiento de contacto;
- exista una auditoria ejecutiva de datasets y una spec cerrada para la ola siguiente de backfill de email.

## Deliverables

- `04_outputs/skilland-knowledge-transfer-engine/email_enrichment_audit.md`
- `04_outputs/skilland-knowledge-transfer-engine/2026-06-07_email-enrichment-backfill-spec_v1.md`
- actualizacion de contexto y proceso en `02_context/`
- actualizacion de estrategia y readmes en `04_outputs/skilland-knowledge-transfer-engine/`

## Scope

- auditar datasets existentes con organizaciones, entidades o contactos utiles para prospeccion B2B;
- cuantificar campos, URLs, emails, trazabilidad y duplicados;
- pausar la spec activa incompatible;
- actualizar estrategia, scoring y definicion de outputs al enfoque `outreach-first / email-first`;
- dejar cerrada la spec de la siguiente ola de enriquecimiento.

## No Scope

- ejecutar nuevo scraping masivo;
- hacer backfill real de emails en esta ola;
- enviar outreach o automatizar CRM;
- deducir emails no publicados;
- ampliar fuentes fuera de las entidades ya extraidas.

## Risks

- El repo arrastra capas historicas pensadas para research o scoring, no para export final de outreach.
- En `interreg_mac` la granularidad actual `partner_in_project` introduce duplicados por entidad y no debe pasar a CRM sin canonizacion.
- Algunos datasets historicos solo guardan URL de listado o telefono, por lo que el backfill exigira una fase previa de normalizacion.
